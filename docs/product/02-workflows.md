# Flujos principales

## Importación

1. Usuario selecciona archivo.
2. Sistema calcula hash antes de transformar.
3. Detector identifica formato y versión.
4. Validador revisa columnas, encoding, timestamps y volumen.
5. Se crea lote inmutable.
6. Parser genera muestras canónicas.
7. Se produce reporte de calidad y cobertura.

## Análisis

1. Usuario selecciona periodo.
2. Motor crea timeline sincronizado.
3. Calcula métricas físicas y estadísticas.
4. Ejecuta reglas versionadas.
5. Correlaciona eventos cercanos.
6. Construye episodios.
7. Asigna severidad y confianza.

## Interpretación IA

1. Motor produce envelope de evidencia.
2. Redactor elimina secretos y datos innecesarios.
3. Proveedor devuelve JSON estructurado.
4. Validador comprueba schema, referencias y cobertura.
5. Un segundo proveedor puede revisar eventos críticos.
6. La UI muestra coincidencias, desacuerdos y límites.

## Revisión técnica

1. Técnico abre episodio.
2. Consulta antes/durante/después.
3. Registra mediciones externas y diagnóstico.
4. Confirma o descarta hipótesis.
5. Registra acción, evidencia y resultado.

## Reporte

1. Selección de episodios.
2. Redacción según audiencia.
3. Inclusión automática de tablas y gráficas.
4. Redacción de seriales según política.
5. Firma/versionado y exportación.
